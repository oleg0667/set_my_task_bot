# FamilyTaskBot — Telegram Task Management Bot

Бот для управления семейными и личными задачами в Telegram. Позволяет создавать задачи с чек-листами, назначать исполнителей, управлять приоритетами и отслеживать статус через инлайн-клавиатуры — всё в одном сообщении без спама.

---

## Установка

### 1. Клонируйте репозиторий и создайте виртуальное окружение

```bash
python -m venv venv
venv\Scripts\activate  # Windows
pip install -r requirements.txt
```

### 2. Настройте конфигурацию

Скопируйте `.env.example` в `.env` и укажите токен бота:

```
BOT_TOKEN=your_bot_token_here
```

Опционально — настройки прокси (для регионов, где Telegram API заблокирован):

```
PROXY_ENABLED=True
PROXY_PROTOCOL=socks5
PROXY_HOST=127.0.0.1
PROXY_PORT=1080
```

### 3. Запустите бота

```bash
venv\Scripts\python -m app.main
```

База данных SQLite (`tasks.db`) создаётся автоматически при первом запуске.

## Команды

### `/task` — Создать задачу

Основная команда для создания задач. Поддерживает:

- **Заголовок** — первая строка после `/task`
- **Чек-лист** — строки, начинающиеся с `- ` или `* `
- **Исполнитель** — `@username` в любой строке
- **Приоритет** — ключевое слово `!low`, `!medium`, `!high` или `!urgent` в заголовке

#### Примеры

Простая задача:
```
/task Купить продукты
- Хлеб
- Молоко
- Яйца
```

Задача с исполнителем и приоритетом:
```
/task !high Выучить до конца недели
* 100 новых финских слов
* Прочитать книгу на финском
 @sherbsherb
```

Результат:
- **Заголовок:** "Выучить до конца недели"
- **Исполнитель:** @sherbsherb (если пользователь есть в базе)
- **Приоритет:** 🟠 Высокий
- **Чек-лист:** 2 пункта

### `/my_task_status` — Проверить статус задачи (для автора)

Показывает полную карточку задачи по её ID.

```
/my_task_status 5
```

Или ответьте (`Reply`) на сообщение с карточкой задачи командой `/my_task_status` — ID подставится автоматически.

### `/my_current_tasks` — Мои текущие задачи (для исполнителя)

Показывает список всех активных задач, назначенных на вас.

```
/my_current_tasks
```

### `/current_task` — Все задачи в группе

Показывает список всех активных задач в текущем чате (группе).

```
/current_task
```

---

### `/start` — Приветствие и диплинки

- Без параметров — показывает приветственное сообщение
- `/start task_<ID>` — диплинк для назначения исполнителя.

**Как работает диплинк:**

1. Создаётся задача с `@username` пользователя, которого нет в базе бота
2. Бот **сам генерирует ссылку** вида `https://t.me/<bot_username>?start=task_<ID>` и отправляет её в чат
3. Вы копируете эту ссылку и отправляете её пользователю в личку
4. Пользователь переходит по ссылке, нажимает **Start**
5. Бот автоматически назначает его исполнителем и показывает карточку задачи

#### Пример сценария: задача для внешнего человека

Вы в группе пишете:

```
/task !high Подготовить отчёт к пятнице
 @alex_ivanov
```

Бот ответит карточкой, где в поле "Исполнитель" будет написано `@alex_ivanov (ожидает регистрации)`, и дополнительно пришлёт сообщение с готовой ссылкой:

```
🔗 Исполнитель @alex_ivanov не зарегистрирован в боте.
Отправьте ему эту ссылку для назначения задачи:
https://t.me/set_my_task_bot?start=task_6
```

Когда @alex_ivanov перейдёт по ссылке и нажмёт Start — задача автоматически назначится на него.

### Автоназначение задач при регистрации

Если пользователь с `@username` уже был упомянут в задаче (как исполнитель), а потом впервые пишет боту в личку или в группе — бот автоматически проверит, есть ли не назначенные задачи с его username, и назначит его исполнителем. То есть **можно просто отправить человеку ссылку на бота**, и все ожидающие его задачи назначатся автоматически.

## Карточка задачи

После создания задачи бот отправляет в чат сообщение с карточкой:

```
📌 ЗАДАЧА #5
От: Oleg
Исполнитель: @sherbsherb (или "Не назначен")
Статус: 📌 Создана
Приоритет: 🟡 Средний

Список:
[ ] 100 новых финских слов
[ ] Прочитать книгу на финском

Обновлено: 2026-07-20 14:19
```

## Инлайн-клавиатуры (кнопки)

Все действия с задачами выполняются через кнопки под карточкой. Сообщение с карточкой **редактируется на месте** — никакого спама.

### Доступные кнопки

| Кнопка | Статус задачи | Действие | Кто может нажать |
|---|---|---|---|
| 🚀 Принять в работу | CREATED | Перевести в статус «В процессе» | Исполнитель (или автор, если исполнителя нет) |
| ✅ Завершить | IN_PROGRESS | Перевести в статус «Завершена» | Исполнитель (или автор, если исполнителя нет) |
| ❌ Провалено | IN_PROGRESS | Перевести в статус «Провалена» | Исполнитель (или автор, если исполнителя нет) |
| 🚫 Отменить | CREATED / IN_PROGRESS | Перевести в статус «Отменена» | Только автор |

Для каждого пункта чек-листа (строки, начинающиеся с `-` или `*`) показываются две отдельные кнопки:

| Кнопка | Действие |
|---|---|
| ✅ [текст] | Отметить пункт как выполненный |
| ❌ [текст] | Отметить пункт как проваленный |

После обработки пункт отображается в карточке как `☑️ [текст]` (выполнен) или `❌ [текст]` (провален).

### Жизненный цикл задачи

```
СОЗДАНА ──(Принять в работу)──> В ПРОЦЕССЕ
  │                                │
  │                                ├──(Завершить)──> ЗАВЕРШЕНА
  │                                │
  │                                └──(Провалено)──> ПРОВАЛЕНА
  │
  └──(Отменить)──> ОТМЕНЕНА
```

## Приоритеты задач

| Ключевое слово | Приоритет | Эмодзи |
|---|---|---|
| `!low` | Низкий | 🟢 |
| `!medium` (по умолчанию) | Средний | 🟡 |
| `!high` | Высокий | 🟠 |
| `!urgent` | Срочно | 🔴 |

Указывается в заголовке задачи (ключевое слово автоматически удаляется из названия):

```
/task !urgent Сдать отчёт до пятницы
- Собрать данные
- Написать выводы
```

## Работа в групповых чатах

Бот автоматически регистрирует участников группы, когда они отправляют сообщение. Это необходимо, чтобы бот мог назначать их исполнителями задач.

**Важно:** Если пользователь ещё ни разу не писал в группе (или боту в личку), его нет в базе — задача создастся без исполнителя. В этом случае используйте диплинк (`/start task_<ID>`).

## Технологии

- **Python 3.11+**
- **aiogram 3** — асинхронный фреймворк для Telegram Bot API
- **SQLAlchemy 2.0 (async)** — ORM
- **SQLite + aiosqlite** — лёгкая встроенная БД
- **Pydantic Settings** — типизированная конфигурация

---

## Права доступа (Business Rules)

- **Автор** задачи может: отменить задачу, завершить (если нет исполнителя)
- **Исполнитель** может: отмечать пункты чек-листа, завершить задачу
- **Любой участник** может: взять не назначенную задачу
- **Только автор** может отменить задачу
